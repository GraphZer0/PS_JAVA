Вы разрабатываете игру, в которой игрок выступает в роли детектива. Его задача — собрать улики, 
найти совпадения с базой данных и определить подозреваемого. Для этого вам нужно реализовать программу, которая:

1. Хранит список найденных улик (например, отпечатки пальцев, записи камер наблюдения) в коллекции Set.
2. Позволяет добавлять новые улики (дубликаты не должны добавляться).
3. Проверяет наличие конкретной улики в наборе.
4. Удаляет улику, если она оказалась ложной.
5. Сравнивает набор найденных улик с базой данных (еще один Set) и находит совпадения.
Функциональные требования:

- Добавление улик:

    Пользователь может добавить новую улику в список найденных улик. Если улика уже существует в наборе, программа должна сообщить об этом.

- Проверка наличия улики:

    Пользователь может ввести название улики, чтобы проверить, есть ли она в списке найденных.
- Удаление улики:

    Если обнаружено, что улика была ошибочной или ложной, пользователь может удалить ее из набора.

- Сравнение с базой данных:

    Программа должна содержать заранее заданный набор «улик из базы данных» (второй Set). 
    Пользователь может запросить сравнение двух наборов (найденных улик и базы данных), чтобы узнать, какие из них совпадают.

- Вывод всех найденных улик:

    Пользователь может запросить вывод всех текущих найденных улик.
